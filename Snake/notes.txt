## Github
Opening the repo, always do git pull first

Adding your changes to the work tree:
1. git status # View the state of the repo
2. git add <some-file> # Stage a file, if you want to add all write a . after add .
3. git commit -m # Commit a file</some-file>, "message"

Then, add your changes to the branch you are currently on:
4. git pull (again, to make sure nobody else has pushed something new)
5. git push

Pushing everything on the branch into main branch:
    git push origin main

    overwriting conflict: 
        git pull --rebase origin main
    if:
        "CONFLICT (content): Merge conflict in <some-file>"
    then: 
        git staus and edit files, then:
            git add <some-file>
            git rebase --continue

            if "what the fuck am i doing this is wierd":
                git rebase --abort

Commit messages:
"Fix: ..."
"Remove:..."
"Add:..."
Always include what file you edited, what functions you created/removed/fixed. 
Do not name anything silly, its supposed to ´be meaningfull and next person have to 
understand what you wrote. This is for tracking progress and create notes on what chanhges
did what in the tree. 

### Branching
have the branches short lived, no point in using them too long and keeping them away from the main
one feature per branch, test feature before merging it

- Main branch:
Do not push anything here that is not working, keep this as clean as possible and only push in things that work.
Its really important that all potential conflicts are resolved before you leave it. DO NOT walk
away from any merge conflicts, fix them befor eyou forget. 

- Rendering:
Only things that has to do with rendering. Files you may edit on this branch: 
- render.py
- app.py
- any factories that will have anything ot do with rendering
- any text files



## Environment

Make sure that the environment is activated before you try to dowload any packages.
Once environemnt is activated in the outter folder, download packages should be no problem

Create an environment for Linux:
    python3 -m venv .venv

Activate environment for Linux:
    source .venv/bin/activate

Install packages:
pip install <package>

Deactivate environment:
deactivate
